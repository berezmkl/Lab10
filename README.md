## 🎯 Мета роботи
Ознайомлення з принципами функціонального тестування RESTful API. Створення локального мок-сервісу на основі `json-server`, формування та відправлення HTTP-запитів через інструмент Hoppscotch, а також вивчення JavaScript Fetch API для реалізації програмних запитів та автоматизації перевірок.

## 🛠️ Технологічний стек та інструменти
* **Мок-сервер:** JSON Server (Node.js)
* **Тест-клієнт:** Hoppscotch (веб-платформа для API-запитів)
* **Автоматизація:** JavaScript Fetch API, асерти Hoppscotch (`pw.expect`)

## 📈 Результати виконання
* **Локальний API:** Розгорнуто та налаштовано мок-сервер для імітації ендпоінтів реального застосунку.
* **Колекції Hoppscotch:** Створено структуровані колекції HTTP-запитів (`GET`, `POST`, `PUT`, `DELETE`). Впроваджено використання змінних середовища (Environment Variables) та передзапитних скриптів (Pre-request scripts) для генерації динамічних параметрів.
* **Тестові скрипти:** Написано автоматичні перевірки відповідей сервера (валідація статус-кодів, структури JSON-тіла та часу відгуку).
* **Програмний Fetch:** Реалізовано JavaScript-функції на базі Node.js, які надсилають асинхронні запити до сервера та коректно обробляють результати чи HTTP-помилки через блоки `try/catch`.

Чим відрізняється тестування API через Hoppscotch від використання Fetch API? Hoppscotch — це спеціалізований інструмент із графічним інтерфейсом (GUI) для ручного тестування, швидкого налагодження, візуального аналізу та створення колекцій запитів. Fetch API — це вбудований безпосередньо в JavaScript програмний інтерфейс (код), який застосовується у веб-додатках або Node.js скриптах для автоматизації відправки HTTP-запитів на рівні логіки програми.

Як обробляються помилки під час роботи з Fetch API? На відміну від багатьох бібліотек (наприклад, Axios), Fetch API не вважає стандартні HTTP-помилки (на кшталт 404 чи 500) причиною для автоматичного відхилення (reject) промісу. Проміс виконається успішно, тому після отримання відповіді необхідно самостійно перевіряти булеву властивість response.ok. Якщо перевірка не пройдена, генерують виняток через throw new Error(), який згодом перехоплюється та обробляється у блоці catch().
